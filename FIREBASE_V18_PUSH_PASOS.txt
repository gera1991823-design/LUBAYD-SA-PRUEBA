FIREBASE V18 PUSH - PASOS OBLIGATORIOS
======================================

1. PLAN DE FIREBASE
-------------------
Cloud Functions requiere que el proyecto esté en el plan Blaze para desplegarse.
En Firebase, pulsa "Actualizar" y vincula una cuenta de facturación.
Puedes configurar alertas y presupuestos en Google Cloud Billing.

2. SUBIR LA WEB
---------------
Sube a GitHub todos los archivos de esta carpeta EXCEPTO node_modules.
Conserva la carpeta functions dentro del repositorio como respaldo, aunque GitHub
Pages no la ejecutará.

3. PUBLICAR REGLAS Y FUNCIÓN DESDE LA PC
----------------------------------------
Instala Node.js y Firebase CLI. Luego abre una terminal dentro de la carpeta V18:

npm install -g firebase-tools
firebase login
cd functions
npm install
cd ..
firebase deploy --only firestore:rules,functions

El proyecto ya está configurado como lubayd-sa en .firebaserc.

4. ACTIVAR CADA CELULAR
-----------------------
- Inicia sesión en la aplicación.
- Abre Mensajes.
- Pulsa "Activar push".
- Acepta el permiso del teléfono.
- Envía un mensaje de prueba desde otra cuenta.

ANDROID
-------
Funciona desde Chrome o desde la PWA instalada. Revisa que Chrome tenga permiso
de notificaciones y que el ahorro de batería no bloquee sus avisos.

iPHONE / iPAD
-------------
- Requiere iOS/iPadOS 16.4 o posterior.
- Abre la web en Safari.
- Usa Compartir > Agregar a inicio.
- Abre la aplicación desde el icono de la pantalla de inicio.
- Dentro de Mensajes pulsa "Activar push" y acepta el permiso.

5. COMPROBAR EL REGISTRO
------------------------
En Firestore aparecerá la colección push_tokens. Cada documento representa un
dispositivo activado por un usuario.

6. PRUEBA
---------
Cierra la PWA o déjala en segundo plano. Desde otro usuario, envía un mensaje.
Debería aparecer una notificación con el nombre del remitente y el texto.

7. SI NO LLEGA
--------------
- Comprueba que la función notifyNewChatMessage esté desplegada.
- Revisa Firebase > Functions > Logs.
- Comprueba que exista un documento del receptor en push_tokens.
- Verifica el permiso de notificaciones del sistema.
- En iPhone, confirma que la web esté instalada en la pantalla de inicio.
